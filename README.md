# Express MVC Server з шаблонізаторами PUG та EJS

Це демонстраційний проект Express сервера, який використовує архітектуру MVC (Model-View-Controller) та два різних шаблонізатори для рендерингу сторінок:
- **PUG** для маршрутів користувачів (`/users`)
- **EJS** для маршрутів статей (`/articles`)

## 🚀 Основні можливості

- ✅ Архітектура MVC (Model-View-Controller)
- ✅ Два шаблонізатори в одному проекті (PUG + EJS)
- ✅ Маршрутизація для користувачів та статей
- ✅ Responsive дизайн з сучасними CSS стилями
- ✅ Тестові дані для демонстрації функціональності
- ✅ Обробка помилок (404 для неіснуючих записів)

## 📋 Вимоги

Перед встановленням переконайтеся, що у вас встановлено:
- **Node.js** (версія 14.x або вище)
- **npm** (зазвичай встановлюється разом з Node.js)

## 📥 Встановлення

### 1. Клонування репозиторію

```bash
git clone https://github.com/Sergei-Monastyretskiy/lessonJS_43.git
cd lessonJS_43
```

### 2. Встановлення залежностей

```bash
npm install
```

Це встановить всі необхідні пакети:
- `express` - веб-фреймворк для Node.js
- `pug` - шаблонізатор PUG
- `ejs` - шаблонізатор EJS
- `nodemon` (dev) - автоматичне перезавантаження сервера при змінах

## 🎯 Запуск проекту

### Звичайний запуск

```bash
npm start
```

### Запуск в режимі розробки (з автоперезавантаженням)

```bash
npm run dev
```

Після запуску сервер буде доступний за адресою: **http://localhost:3000**

## 🛣️ Маршрути та функціональність

### Головна сторінка

- **GET /** - Головна сторінка з описом проекту
  - Шаблонізатор: PUG
  - Відображає вітальну сторінку з посиланнями на розділи

### Маршрути користувачів (PUG)

#### 1. Список користувачів
- **URL:** `GET /users`
- **Опис:** Відображає список всіх користувачів
- **Шаблонізатор:** PUG
- **Шаблон:** `views/users/index.pug`
- **Дані:** 5 тестових користувачів з полями:
  - `id` - унікальний ідентифікатор
  - `name` - ім'я користувача
  - `email` - електронна пошта
  - `age` - вік
  - `city` - місто

**Приклад використання:**
```
http://localhost:3000/users
```

#### 2. Деталі користувача
- **URL:** `GET /users/:userId`
- **Опис:** Відображає детальну інформацію про конкретного користувача
- **Шаблонізатор:** PUG
- **Шаблон:** `views/users/detail.pug`
- **Параметри:**
  - `:userId` - ID користувача (число)
- **Обробка помилок:** Якщо користувача не знайдено, відображається повідомлення про помилку

**Приклади використання:**
```
http://localhost:3000/users/1
http://localhost:3000/users/2
http://localhost:3000/users/999  (помилка 404)
```

#### 3. Інші API маршрути користувачів
- **POST /users** - Створити нового користувача (поки повертає текстове повідомлення)
- **PUT /users/:userId** - Оновити користувача (поки повертає текстове повідомлення)
- **DELETE /users/:userId** - Видалити користувача (поки повертає текстове повідомлення)

### Маршрути статей (EJS)

#### 1. Список статей
- **URL:** `GET /articles`
- **Опис:** Відображає список всіх статей
- **Шаблонізатор:** EJS
- **Шаблон:** `views/articles/index.ejs`
- **Дані:** 5 тестових статей з полями:
  - `id` - унікальний ідентифікатор
  - `title` - заголовок статті
  - `author` - автор
  - `content` - зміст статті
  - `date` - дата публікації
  - `views` - кількість переглядів

**Приклад використання:**
```
http://localhost:3000/articles
```

#### 2. Деталі статті
- **URL:** `GET /articles/:articleId`
- **Опис:** Відображає повний зміст конкретної статті
- **Шаблонізатор:** EJS
- **Шаблон:** `views/articles/detail.ejs`
- **Параметри:**
  - `:articleId` - ID статті (число)
- **Обробка помилок:** Якщо статтю не знайдено, відображається повідомлення про помилку

**Приклади використання:**
```
http://localhost:3000/articles/1
http://localhost:3000/articles/3
http://localhost:3000/articles/999  (помилка 404)
```

#### 3. Інші API маршрути статей
- **POST /articles** - Створити нову статтю (поки повертає текстове повідомлення)
- **PUT /articles/:articleId** - Оновити статтю (поки повертає текстове повідомлення)
- **DELETE /articles/:articleId** - Видалити статтю (поки повертає текстове повідомлення)

## 🔧 Конфігурація шаблонізаторів

### Налаштування в `src/app.js`

```javascript
// PUG як основний шаблонізатор
app.set('view engine', 'pug');
app.set('views', path.join(__dirname, '../views'));

// Реєстрація EJS як додаткового шаблонізатора
app.engine('ejs', require('ejs').renderFile);

// Статичні файли
app.use(express.static(path.join(__dirname, '../public')));
```

### Використання шаблонів

**PUG (для користувачів):**
```javascript
res.render('users/index', { users, title: 'Список користувачів' });
```

**EJS (для статей):**
```javascript
res.render('articles/index.ejs', { articles, title: 'Список статей' });
```
